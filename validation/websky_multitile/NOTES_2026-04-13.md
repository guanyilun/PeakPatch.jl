# Validation Notes — 2026-04-13 — Multi-Tile (ntile=2)

## Goal
Validate PeakPatch.jl multi-tile pipeline against Fortran multi-tile MPI,
and verify internal consistency (Julia multi-tile = Julia single-tile on
same total grid).

## Configuration

### Geometry
Same tile size as the 256³ single-tile validation, with ntile=2:
- nmesh = 256, nbuff = 22, ntile = 2
- nsub = 256 - 44 = 212
- cellsize = 512 / 212 = 2.4151 Mpc/h
- dcore_box = 512.0 Mpc/h (per-tile core)
- dL_box = 618.264 Mpc/h (per-tile total, = Julia `boxsize`)
- Total grid: N = 212×2 + 44 = 468
- Total box: 1130.264 Mpc/h (= 468 × cellsize)
- Total core volume: (2 × 212)³ = 424³ cells = (1024 Mpc/h)³

### Cosmology (Planck 2018)
- Omx = 0.2645, OmB = 0.0493, Omvac = 0.6862, h = 0.6735
- Same as 256³ and 1024³ validation runs

### Physics
- seed = 13579, wsmooth = 1 (top-hat), ilpt = 2 (2LPT), rmax2rs = 0.0
- z_out = 0.0, ioutshear = 0
- 16 log-spaced filters (same as 256³ validation)
- Collapse table: HomelTab_matched.dat

## Three Runs

### 1. Fortran MPI (ntile=2)
- 4 MPI ranks, 8 OMP threads each (N=468 must be divisible by nranks: 468/4=117)
- Initial attempt with 8 ranks failed: `mod(n, ntasks) != 0` in Fortran FFTW slab decomposition
- n1=256 (recompiled from 1024)
- nlx=nly=nlz=2 in hpkvd_params.bin
- Same filter bank and collapse table as 256³ validation
- PBS: `run_fortran.pbs`

### 2. Julia multi-tile (ntile=2)
- Serial `run_multitile(cfg; ntile=2)` with 32 threads
- `use_lcg=true, fortran_compat=true` for Fortran-compatible RNG and Float32
- Config: `config_multitile.toml`
- PBS: `run_julia_multitile.pbs`

### 3. Julia single-tile (nmesh=468)
- `run_tile(cfg)` with nmesh=468, ntile=1
- Same total 468³ grid, core=424³=2×212³ (same cells as ntile=2 tiles)
- `use_lcg=true, fortran_compat=true`
- Config: `config_singletile.toml`
- PBS: `run_julia_singletile.pbs`

### Why runs 2 and 3 should match
Both generate the same GRF on a 468³ grid with seed=13579. The multi-tile
run (ntile=2) finds peaks in two 212³ core regions per dimension; the
single-tile run (nmesh=468) finds peaks in one 424³ core region. Since
424 = 2×212, the core cells are identical — no gap, no overlap. Smoothing
is performed on the full 468³ grid in both cases. The only potential
difference is floating-point ordering in the peak list.

## Expected Results
- Julia ntile=2 vs Julia ntile=1: identical halo count (same field, same core cells)
- Julia ntile=2 vs Fortran ntile=2: ~0.4-1.4% halo count difference (same as single-tile validation, from Float32 rounding)

## Peak Count Estimate
From the 256³ single-tile run (same tile size):
- ~80k peaks, ~68k halos per 212³ core
- With 8 tiles (ntile=2): ~8×80k = 640k peaks, ~8×68k = 544k halos total
- Per-tile: same ~80k peaks → well under 2M limit

## Issues Encountered

1. **Fortran MPI slab decomposition**: Initial submission with 8 MPI ranks failed
   because the total grid N=468 is not divisible by 8 (468/8=58.5). The Fortran
   FFTW MPI slab decomposition requires `mod(N, ntasks) == 0`. Fixed by using
   4 ranks (468/4=117). The 8 tiles are distributed round-robin: 2 tiles per rank.

2. **CLI driver missing use_lcg/fortran_compat passthrough**: `bin/peakpatch.jl`
   was not passing `use_lcg` and `fortran_compat` to `run_multitile()` (only to
   `run_tile()`). Fixed before submission.

### Interpretation

**Julia internal consistency (ntile=2 vs ntile=1):**
Essentially perfect — 1 peak and 1 halo difference out of 651k/550k. Both runs
generate the same 468³ GRF with the same LCG seed, apply the same smoothing on
the full grid, and search the same core cells (424³ = 2×212³). The single
differing halo is at a tile boundary where floating-point evaluation order
in peak finding differs.

**Julia vs Fortran (ntile=2):**
Julia has 0.96% more peaks but 0.83% fewer halos. The peak surplus is consistent
with the single-tile validation (~1-2% from Float32 threshold jitter). The halo
count being slightly lower in Julia (vs slightly higher in single-tile 256³) is
within the expected range: shell analysis amplifies or suppresses threshold-edge
peaks depending on their local density profile, and the sign of the net effect
can vary with geometry and random realization. The key metric — **sub-percent
halo agreement (0.83%)** — validates the multi-tile implementation.
