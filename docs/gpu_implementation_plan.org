#+STARTUP: overview

* Motivation

The multi-resolution approach reduces PeakPatch's memory requirement from
O(N^3) (global FFT) to O(nmesh^3) per tile (~40 GB at 6144^3). This fits
on a single GPU (A100 80 GB or H100 80 GB). The dominant compute cost per
tile is 3D FFTs and radial shell analysis, both of which map well to GPU.

At 6144^3 with ntile=9, there are 729 tiles (~341 active after lightcone
pruning). Each tile is fully independent — no inter-tile communication.
This is embarrassingly parallel across GPUs.

*No GPU cosmological IC generator or peak-patch halo finder exists publicly.*
PeakPatch.jl would be first-of-kind (as of 2026).

Key insight from CompaSO: their spherical overdensity algorithm searches
outward from density peaks to find collapse radius — structurally identical
to PeakPatch's =analyse_peak=. They run on Summit GPUs during Abacus N-body.

The entire per-tile pipeline runs on GPU. Only the coarse grid FFT (M=24,
trivial) and final catalog I/O remain on CPU.

* Memory Budget per GPU

For N=6144, nmesh=768, isolated FFT pads to 1536^3:

| Component                         | Size (Float32) | Notes                     |
|-----------------------------------+----------------+---------------------------|
| Padded FFT (1536^3, in-place)     | 14.5 GB        | Shared real/complex buffer |
| Residual noise (768^3)            | 1.8 GB         | Input to isolated FFT     |
| delta_tile (768^3)                | 1.8 GB         | Output, kept              |
| psi_tile x3 (768^3 each)         | 5.4 GB         | 1LPT displacements        |
| psi2_tile x3 (768^3 each)        | 5.4 GB         | 2LPT displacements        |
| lapd_tile (768^3)                 | 1.8 GB         | Laplacian                 |
| delta_tile_k (rfft, 385x768x768) | 1.8 GB         | For smoothing/peak find   |
| Smoothed field (768^3)            | 1.8 GB         | Per-filter temporary      |
| Masks (768^3, Int8)               | 0.45 GB        | Peak exclusion            |
| Shell offset table                | ~0.5 GB        | Precomputed (di,dj,dk)    |
| Collapse table (3D)               | <0.01 GB       | Fits in texture memory    |
|-----------------------------------+----------------+---------------------------|
| *Peak total*                      | *~35 GB*       | Fits 40 GB A100           |

With careful memory reuse (free padded buffer after each FFT, reuse for
next field), peak can be reduced to ~25 GB.

* Implementation Phases

** Phase 1: GPU-accelerated isolated FFT [0/4]

The most straightforward change. Replace CPU FFTW with GPU cuFFT for the
isolated convolution.

*** TODO Port =_isolated_convolve= to GPU
- Input: residual noise (768^3 Float32) on CPU
- Copy to GPU, zero-pad to 1536^3
- cuFFT forward (rfft)
- Apply P(k) kernel (element-wise GPU kernel)
- cuFFT inverse (irfft)
- Extract tile region, keep on GPU for next phases

#+begin_src julia
using CUDA, CUDA.CUFFT

function _isolated_convolve_gpu(noise::CuArray{Float32,3}, pk_table_d::CuVector,
                                 log_k_min::Float32, d_log_k::Float32,
                                 dx::Float64, n::Int;
                                 kernel_fn_id::Int=0, nshell::Int=0)
    n_input = size(noise, 1)
    n2 = 2 * n_input

    # Zero-pad on GPU
    padded_d = CUDA.zeros(Float32, n2, n2, n2)
    padded_d[1:n_input, 1:n_input, 1:n_input] .= noise

    # Forward FFT (cuFFT, plan cached externally)
    padded_k_d = rfft(padded_d)
    CUDA.unsafe_free!(padded_d)  # free real buffer immediately

    # Apply transfer function — fused CUDA kernel
    dk = Float32(2π / (n2 * dx))
    @cuda threads=256 blocks=cld(length(padded_k_d), 256) _transfer_kernel!(
        padded_k_d, pk_table_d, log_k_min, d_log_k, dk, Int32(n2), Int32(kernel_fn_id))

    # Inverse FFT
    result_d = irfft(padded_k_d, n2)
    CUDA.unsafe_free!(padded_k_d)

    # Extract tile region (stays on GPU)
    s1 = nshell + 1; s2 = nshell + n
    return result_d[s1:s2, s1:s2, s1:s2]
end
#+end_src

** Phase 2: GPU shell analysis (critical path) [0/6]

*This is the most important phase for overall performance.* Once FFTs
move to GPU (~2s/tile), shell analysis (~120s/tile on CPU) completely
dominates. Porting shell analysis to GPU transforms the pipeline from
hours to minutes.

*** Design: Block-per-peak kernel (CompaSO-inspired)

The radial shell analysis problem: for each of ~100k peaks, walk outward
in concentric shells, gathering field values and checking collapse criteria.

**** Why block-per-peak

| Strategy          | Threads/peak | Pros                | Cons                    |
|-------------------+--------------+---------------------+-------------------------|
| Thread-per-peak   | 1            | Simple              | Serial gather, very slow |
| Warp-per-peak     | 32           | Warp shuffles       | Too few for large shells |
| *Block-per-peak*  | 128-256      | Full block reduce   | *Best balance*          |

At radius r=30, a shell has ~11,310 cells (4πr²). With 128 threads per
block, each thread handles ~88 cells — good GPU utilization. Warp shuffles
within each warp, then shared memory reduction across warps.

**** Kernel pseudocode

#+begin_src c
// One thread block (128 threads) per peak
__global__ void shell_analysis_kernel(
    const float* __restrict__ delta,    // 768^3, use __ldg for texture cache
    const float* __restrict__ psi_x,    // 768^3
    const float* __restrict__ psi_y,    // 768^3
    const float* __restrict__ psi_z,    // 768^3
    const int3* __restrict__ shell_offsets,  // precomputed (di,dj,dk) per shell
    const int*  __restrict__ shell_start,    // cumulative offset into shell_offsets
    const int*  __restrict__ shell_count,    // number of cells per shell radius
    const PeakCandidate* peaks,
    HaloResult* results,
    int n_peaks, int nmesh, int max_shells)
{
    int peak_id = blockIdx.x;
    if (peak_id >= n_peaks) return;

    int3 center = peaks[peak_id].ijk;

    // Cumulative accumulators (in registers, first thread)
    float cum_delta = 0, cum_mass = 0;
    float cum_psi[3] = {0, 0, 0};
    float cum_strain[6] = {0};  // symmetric 3x3
    int best_r = 0;
    float best_rthl = 0;

    __shared__ float sdata[256];  // for block reduction

    for (int r = 1; r <= max_shells; r++) {
        int n_cells = shell_count[r];
        int offset = shell_start[r];

        // Cooperative gather: each thread handles a slice of cells
        float local_delta = 0, local_px = 0, local_py = 0, local_pz = 0;
        for (int i = threadIdx.x; i < n_cells; i += blockDim.x) {
            int3 d = shell_offsets[offset + i];
            int gi = (center.x + d.x) % nmesh;  // periodic within tile
            int gj = (center.y + d.y) % nmesh;
            int gk = (center.z + d.z) % nmesh;
            int idx = gi + gj * nmesh + gk * nmesh * nmesh;

            local_delta += __ldg(&delta[idx]);  // texture cache read
            local_px    += __ldg(&psi_x[idx]);
            local_py    += __ldg(&psi_y[idx]);
            local_pz    += __ldg(&psi_z[idx]);
        }

        // Block-level reduction via warp shuffle + shared memory
        float total_delta = block_reduce_sum(local_delta, sdata);
        float total_px    = block_reduce_sum(local_px, sdata);
        // ... similarly for py, pz, strain components ...

        // First thread updates cumulative values and checks collapse
        if (threadIdx.x == 0) {
            cum_delta += total_delta;
            cum_psi[0] += total_px;
            cum_psi[1] += total_py;
            cum_psi[2] += total_pz;
            cum_mass = cum_delta / (4.0f/3.0f * M_PI * r*r*r);

            // Check collapse criterion (lookup in collapse table texture)
            float e_v, p_v;
            compute_ellipticity(cum_strain, &e_v, &p_v);
            float zvir = tex3D(collapse_table, cum_mass, e_v, p_v);

            if (zvir > 0 && r > best_r) {
                best_r = r;
                best_rthl = r;
                // ... store other halo properties ...
            }

            // Early termination: if density too low, no larger shell will collapse
            sdata[0] = (cum_mass < threshold_low) ? 1.0f : 0.0f;
        }
        __syncthreads();
        if (sdata[0] > 0) break;  // all threads exit together
    }

    // First thread writes result
    if (threadIdx.x == 0 && best_r > 0) {
        results[peak_id] = make_halo_result(peaks[peak_id], best_r, ...);
    }
}
#+end_src

**** =block_reduce_sum= implementation

#+begin_src c
__device__ float block_reduce_sum(float val, float* sdata) {
    // Intra-warp reduction via shuffle
    for (int offset = 16; offset > 0; offset >>= 1)
        val += __shfl_down_sync(0xffffffff, val, offset);

    // First thread in each warp writes to shared memory
    int lane = threadIdx.x % 32;
    int warp = threadIdx.x / 32;
    if (lane == 0) sdata[warp] = val;
    __syncthreads();

    // First warp reduces across warps
    val = (threadIdx.x < blockDim.x / 32) ? sdata[threadIdx.x] : 0;
    if (warp == 0) {
        for (int offset = 16; offset > 0; offset >>= 1)
            val += __shfl_down_sync(0xffffffff, val, offset);
    }
    return val;  // valid only for threadIdx.x == 0
}
#+end_src

*** TODO Precompute shell offset tables
The shell cells at each radius are known at init time (same for all peaks).
Precompute and upload to GPU once:

#+begin_src julia
function _build_shell_offsets_gpu(max_r)
    offsets = Tuple{Int32,Int32,Int32}[]
    starts = Int32[0]
    counts = Int32[]
    for r in 1:max_r
        r2_min = (r - 1)^2
        r2_max = r^2
        shell = Tuple{Int32,Int32,Int32}[]
        for dk in -r:r, dj in -r:r, di in -r:r
            d2 = di^2 + dj^2 + dk^2
            r2_min < d2 <= r2_max || continue
            push!(shell, (Int32(di), Int32(dj), Int32(dk)))
        end
        append!(offsets, shell)
        push!(counts, Int32(length(shell)))
        push!(starts, Int32(length(offsets)))
    end
    return CuArray(reinterpret(Int32, offsets)),  # flat (di,dj,dk) triples
           CuArray(starts), CuArray(counts)
end
#+end_src

For max_r=50: ~523k offset entries, ~6 MB. Fits easily in GPU memory.

*** TODO Sort peaks by Morton/Z-order for cache efficiency
Consecutively-launched blocks should access overlapping grid regions to
maximize L2 cache reuse (A100 has 40 MB L2):

#+begin_src julia
function _morton_sort_peaks!(peaks)
    # Interleave bits of (i, j, k) to form Morton code
    morton(p) = _interleave_bits(p.i >> 3, p.j >> 3, p.k >> 3)
    sort!(peaks; by=morton)
end
#+end_src

Shifting by 3 bits (dividing by 8) groups peaks into 8^3 spatial blocks,
so peaks within the same L2 cache footprint are processed together.

*** TODO Port =analyse_peak= field access to GPU
The full =analyse_peak= function accesses multiple fields (delta, psi×3,
psi2×3, laplacian, strain). Each shell gather needs ~10 field reads per
cell. Options:

- *Separate kernel per field*: simpler but more kernel launches
- *Fused kernel gathering all fields*: fewer launches, higher register
  pressure, but better memory locality (same cell index for all fields)
- *Struct-of-arrays layout*: interleave fields for coalesced access
  =fields[field_id][x,y,z]= vs =fields[x,y,z,field_id]=

Recommended: fused kernel with all fields passed as separate pointers
(current layout). Register pressure is manageable at 128 threads/block
with ~10 float accumulators per thread.

*** TODO Handle early termination across block
The CPU code has complex early-termination logic in =analyse_peak=
(collapse criterion, radial cutoff). On GPU:

- First thread checks criterion after each shell reduction
- Writes flag to shared memory
- All threads read flag after =__syncthreads()= and break together
- This is the pattern shown in the pseudocode above

@inline function _threefry_gpu(key0::UInt64, key1::UInt64,
                                ctr0::UInt64, ctr1::UInt64)
    # 20-round Threefry2x (same as Random123.jl)
    ks2 = 0x1BD11BDAA9FC1A22  # skein parity constant
    ks0 = key0; ks1 = key1; ks2 ⊻= ks0 ⊻ ks1
    x0 = ctr0 + ks0; x1 = ctr1 + ks1
    # 20 rounds of mix operations...
    return x0, x1
end

* Julia Implementation Notes

** Use =@cuda= kernels, not KernelAbstractions.jl

Shell analysis needs warp-level shuffle intrinsics (=CUDA.shfl_down_sync=)
and explicit shared memory. KernelAbstractions.jl does not expose these.
Use raw =@cuda= kernels for shell analysis and peak finding.

For FFT and element-wise operations, CuArray broadcasting and CUFFT
work out of the box without custom kernels.

** CUDA.jl performance vs native CUDA C

Benchmarks show negligible overhead for compute-bound kernels. The Julia
GPU compilation pipeline (GPUCompiler.jl -> LLVM -> PTX -> cubin) produces
code comparable to nvcc. Main overhead: first kernel launch (JIT), ~1-5s.
Subsequent launches are cached.

** Package extension structure

#+begin_example
src/
  MultiResolution.jl          # Existing CPU implementation (unchanged)
ext/
  CUDAExt.jl                  # GPU extension (loaded when CUDA.jl available)
    _isolated_convolve_gpu    # GPU isolated FFT
    _generate_residual_gpu    # GPU Threefry noise + residual
    _find_peaks_gpu           # GPU peak finding
    _shell_analysis_gpu       # GPU radial shell analysis (block-per-peak)
    run_multitile_split_gpu   # Full GPU tile pipeline
#+end_example

The GPU code lives in a package extension (like MPIExt.jl), so CUDA.jl
is only loaded when available. The CPU path remains the default.

* Key Risks and Mitigations

| Risk                                 | Impact | Mitigation                             |
|--------------------------------------+--------+----------------------------------------|
| Shell analysis register pressure     | High   | Limit accumulators, split into sub-kernels |
| Threefry reproducibility on GPU      | Medium | Port exact bitwise ops to CUDA         |
| L2 cache thrashing (random access)   | Medium | Morton-sort peaks, use __ldg intrinsic |
| cuFFT plan creation overhead         | Low    | Cache plan, reuse across tiles         |
| Julia CUDA compilation latency       | Medium | Precompile, use function barriers      |
| Block-per-peak load imbalance        | Low    | 100k peaks >> SMs, natural balancing   |
